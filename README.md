### ⚔️ Operational Standards

My work is distinguished by how I handle the things users *don't* see immediately:

*   **Failure Modeling:** I model failure scenarios before happy paths.
*   **Maintainability:** I optimize for 3-year maintainability, not 3-week velocity.
*   **Complexity Reduction:** I remove code and reduce abstractions whenever possible.
*   **Security Integration:** Security is embedded in the design phase, not patched in before release.
